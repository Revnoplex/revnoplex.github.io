Data Collection Policy

Revnobot will collect and store certain types of data for developer/debugging purposes and for servers to moderate.

Developer/debugging data is only stored locally on the bot's host machine.

Server moderation data is stored in channels setup by the admin of a server (if setup by server admins).
See the moderation help menu in the Bot for more information on how to set this up

The types of Developer/Debugging data collected include the following:

  • Server Names and IDs
  • Server Owner Usernames and IDs
  • Invite Access to servers if granted invite permission
  • Messages: Revnobot will only collect messages related to command execution. What is collected:
    • Messages that start with the bot's prefix
    • Messages that the bot sent.
    • Slash, user and message commands executed to the bot.
    • The names of the command passed to the bot
    • DMs sent to the bot

The types of Server Moderation data collected (if setup) include the following:

  • When members join or leave the server (username, etc.)
  • Messages (before and after) when a server member edits a message
  • Messages when a server member deletes a message
  • Messages when a user sends a message with a url flagged as malicious
  • Channels (names, channel topic, other channel settings) when a channel is created, edited or deleted)
  • Server Settings when changed
  • Server Emojis when added, edited, or removed
  • Server Roles when added or deteted
  • When members are banned (username, etc.)
  • Server Profiles (nickname, etc.) of members when changed
  • Server Roles members have when given or removed
  • When a member joins or leaves a voice channel (channel name, username, etc.)
  • Invites when a member creates or deletes an invite
  • Pinned message when a member pins or unpins a message
